


# 🌟 AIOps Self-Healing Kubernetes Platform

> 🚨 What if your Kubernetes cluster could *think*, *diagnose itself*, and *heal automatically* — even on a ₹20,000 laptop?

This project is my answer.

---

## 🔥 The Problem

Every DevOps engineer knows this pain:

* ❌ Pods crash at 2 AM
* ❌ You check logs manually
* ❌ You Google errors endlessly
* ❌ You rollback blindly
* ❌ You lose time, sleep, and confidence

👉 Production systems don’t wait for humans.

So I asked:

> “What if Kubernetes could fix itself… intelligently?”

---

## 🧠 The Idea

Build a **closed-loop self-healing system**:

```
Detect → Understand → Decide → Heal → Learn
```

But with a twist:

💡 The “brain” is a **local AI model running on a laptop**
💡 No OpenAI API
💡 No cloud dependency
💡 No expensive infra

Just pure open-source intelligence.

---

## ⚙️ What I Built

A full production-style AIOps platform:

### 🧩 CI/CD Layer

* GitHub Actions builds microservices
* Pushes images to GHCR automatically

### ☸️ GitOps Layer

* ArgoCD syncs everything from Git
* No manual kubectl deployments

### 📊 Observability Stack

* Prometheus → Metrics
* Grafana → Dashboards
* Loki → Logs

### 🤖 AI Brain (Core Innovation)

* FastAPI AI Agent
* Runs locally via Ollama
* Uses lightweight model:

  * `qwen2.5:0.5b-instruct-q4_0`

### 🛠️ Self-Healing Engine

* Detects:

  * CrashLoopBackOff
  * OOMKilled
  * Pod failures
* Sends context → AI
* Gets root cause + fix
* Executes recovery automatically

### 💬 ChatOps Layer

* Discord bot interface:

  * `!health`
  * `!analyze`
  * `!heal`

## 💻 Built for LOW-END Hardware (Yes, really)

Most AI systems assume GPUs.

This one runs on:

* 💻 Intel Celeron (no AVX support)
* 🧠 4GB RAM laptop
* 🐧 Ubuntu 24.04
* ⚡ Entire stack < 3GB RAM

### Why it works

* Uses **quantized LLM (0.5B model)**
* CPU-safe inference
* Optimized for edge environments

---

## 🚀 Quick Start (3 Steps Only)

```bash
# 1. Start cluster
minikube start --cpus=2 --memory=4096

# 2. Deploy platform
helm install aio-self-healing ./charts/aio-self-healing -n aiops --create-namespace

# 3. Break something 😈
kubectl apply -f examples/crashy-app.yaml
```

Then watch magic happen:

```bash
kubectl logs -f deployment/aio-self-healing-healer-controller -n aiops
```
